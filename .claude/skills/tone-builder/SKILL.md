# Tone Builder — 곡 → GP-150 패치

곡 이름을 받아 Cort G250(HSS) + Valeton GP-150 기준 풀 시그널 체인 패치를 만든다.

## 먼저 읽을 것

매번 추천 전에 다음 레퍼런스를 읽는다:

1. `reference/gear-profile.md` — 내 기타/이펙터 고정 스펙, 픽업 선택 가이드, 시그널 체인 규칙
2. `reference/gp150-amps.md` — 앰프 모델 ↔ 캐릭터 매핑 (매칭의 핵심)
3. `reference/gp150-effects.md` — 이펙트 블록 캐릭터

## 동작 순서

### 1. 곡 톤 조사
곡 이름/아티스트를 받으면 원곡의 기타 톤을 조사한다 (필요시 웹 검색):
- 쓰인 앰프/이펙트, 장르
- 클린 / 크런치 / 하이게인 정도
- 특징적 이펙트 (코러스, 딜레이, 와우 등)
- 곡 안에서 사운드가 바뀌는 구간 (벌스 클린 → 솔로 부스트 등)

### 2. 앰프 매칭
조사한 캐릭터를 `gp150-amps.md`와 대조해 가장 가까운 GP-150 앰프 모델 1~2개 선택.
근거(왜 이 앰프인지)를 짧게 남긴다.

### 3. 풀 체인 구성
`gear-profile.md`의 시그널 체인 규칙에 따라 구성하고 **구체적 노브 값**을 제시:
- 앰프: 게인, Bass/Mid/Treble, Presence, Level
- 캐비닛/IR
- 이펙트 블록별 핵심 파라미터 (딜레이 타임·피드백·믹스, 리버브 믹스 등)
- G250 픽업 포지션 (gear-profile의 픽업 가이드 사용)

### 4. 스위칭 플랜 (선택)
곡 안에서 사운드 전환이 필요하면 풋스위치 2개 + 익스프레션 페달로 현실적 전환안 작성.
전환이 없으면 이 섹션 생략.

### 5. 저장
- `patches/<artist>-<song>.md` 생성 (아래 템플릿)
- `patches/INDEX.md` 표에 한 줄 추가
- 파일명은 소문자-하이픈 (예: `oasis-wonderwall.md`)

## 불확실성 처리 (중요)

- 곡 톤을 못 찾거나 확신이 낮으면 **추측으로 노브 값을 채우지 않는다.**
- "이 부분은 확신 낮음"이라고 표시하고 대안(앰프 후보 2개 등)을 같이 제시한다.
- GP-150 풋스위치는 물리적으로 2개뿐 — 그 제약 안에서 전환안을 짠다.

## 패치 파일 템플릿

```markdown
# <Artist> – <Song>

- **장르/톤 요약**: <한 줄>
- **확신도**: 높음 / 보통 / 낮음 + 근거

## 시그널 체인
기타 → [블록1] → [블록2] → ... → 출력

## 모듈 설정
### 앰프: <GP-150 모델명> (원본: <실제 앰프>)
- Gain / Bass / Mid / Treble / Presence / Level
### 캐비닛/IR
- <IR 또는 팩토리 캐비닛>
### 이펙트
- <블록>: 핵심 파라미터 값

## G250 픽업
- 포지션: <5-way 위치 / 코일스플릿 여부>

## 스위칭 플랜 (선택)
- 기본: ...
- 풋스위치 A → ...
- 풋스위치 B → ...
- 익스프레션 페달 → ...

## 왜 이 앰프인지 & 연주 팁
- ...
```

## 검증

새 패치는 실제로 GP-150에 입력해 귀로 듣고 피드백하는 것이 진짜 검증이다.
사용자 피드백이 오면 해당 패치 파일과(원인이 데이터면) 레퍼런스를 함께 보정한다.
