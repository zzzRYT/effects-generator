# PRD — signal-chain-view (곡 상세 시그널 체인 렌더러)

- **Feature slug**: signal-chain-view
- **Brainstorm/설계**: docs/plans/2026-06-21-signal-chain-view-design.md
- **상태**: DRAFT

## 무엇 (What)
`/songs/[slug]` 정적 라우트에서 한 곡의 패치를 GP-150 기계 패널처럼 보여준다.
곡의 **모든 변주를 세로로 나열**하고, 각 변주는 signal_chain을 가로 흐름으로 그린
범용 블록-체인 렌더러(블록=하드웨어 슬롯, 노브=LCD 텍스트 셀)와 스위칭 플랜 섹션으로 구성된다.
렌더러는 `block.type`만 보고 그린다.

## 왜 (Why) / 누구를 위해
지금은 패치가 md/타입 상수로만 존재해 **눈으로 톤을 읽을 수 없다.** GP-150 유저(나·밴드)가
곡을 골라 "어떤 블록을 어떤 순서로, 어떤 노브 값으로" 세팅하는지 한눈에 보고 그대로 따라
세팅할 수 있어야 한다. 이게 소비 레이어의 핵심 가치("whoa") — 보여줄 게 생긴다.

## 수용 기준 (측정 가능 — 테스트로 검증)
- [ ] AC1: 곡 상세에서 변주의 signal_chain 모든 block이 **md 순서대로** 렌더된다.
- [ ] AC2: 곡의 **모든 변주**가 세로로 나열되고, 각 변주는 자기 데이터만 렌더(혼합 0).
- [ ] AC3: 각 block에 타입 약어 텍스트(OD/AMP/CAB…)가 색 배지와 **병기**된다(색만으로 의미 전달 금지).
- [ ] AC4: 노브가 data-contract §2 형식으로 렌더된다 — 단위 있음 `Time: 640ms`, 없음 `Gain: 5.5 (0–10)`, 부동소수 보존.
- [ ] AC5: `enabled=false` 블록은 opacity 0.30 + grayscale + 'OFF' 라벨 + `aria-disabled`로 표시되되 **노브 값은 전부 보인다**.
- [ ] AC6: `footswitch` 블록은 'A'/'B' 배지 + aria-label, 같은 키 블록은 시각 그룹화. 무스위치 블록은 배지 영역 clean.
- [ ] AC7: switching 메타가 signal_chain과 **분리된** "스위칭 플랜" 섹션에 description + `(N개: 모델…)`로 표시.
- [ ] AC8: `knobs=[]` 블록은 "노브 없음" 명시, crash 0.
- [ ] AC9: 320/768/1024/1440에서 오버플로 없음.
- [ ] AC10: 빌드가 정적(SSG) 유지 — 모든 곡 라우트 `○ Static`, 런타임 페치 0.

## 비목표 (Non-goals)
- 변주 비교 탭/나란히 보기 → #2(variation-compare).
- 곡 목록/검색/진입 네비 → #3(song-index). (이번엔 홈에서 임시 링크만)
- 노브 회전 다이얼 그래픽 → 채택 안 함(LCD 텍스트 셀로 확정).
- 인터랙티브 토글/편집 → 읽기 전용(프로젝트 헌법).
- 라이트 테마/테마 토글 → 다크 단일.
