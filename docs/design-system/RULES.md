# Tone Forge 디자인 규칙 (전역 · 권위 문서)

> **적용 범위:** `web/` 의 모든 UI. 새 화면·새 컴포넌트·기존 화면 수정 전부.
> **권위 순서:** Claude Design 프로젝트 "Tone Forge Design System"
> (`projectId: d8ed3286-46e5-40e7-a0b3-b5b7d815bf30`) → 이 문서 → 각 `*.module.css`.
> 로컬 미러 = `docs/design-system/tokens/*.css` (원본 그대로).
> 코드 단일 출처 = `web/lib/tokens.css` (토큰) + `web/lib/hardware.css` (프리미티브).

## 0. 한 줄 요약

**투박한 하드웨어 스큐어모피즘.** 진짜 플로어 멀티이펙터를 조작하는 느낌.
니어블랙 섀시 · 앰버 진공관 글로우 단일 악센트 · 실크스크린 대문자 라벨 ·
융기 vs 함몰 릴리프 · 정직한 실제 기어색(**네온 금지**).

## 1. 색 — 어길 수 없는 규칙

| 역할 | 토큰 | 쓰는 곳 |
| --- | --- | --- |
| 브랜드 악센트 (**유일**) | `--accent` (앰버 `#f59422`) | CTA, 포커스 링, 활성 탭, 링크 |
| 표면 | `--chassis-900…100`, `--silkscreen` | 배경·패널·웰·헤어라인 |
| 상태 = LED | `--led-green/amber/red/blue` | 점등·클립·바이패스·동기 |
| LCD 리드아웃 | `--lcd-bg` / `--lcd-text` | 수치 표시창 |
| 블록 타입 (**길찾기 전용**) | `--color-od/amp/cab/dly/rvb/mod/util` | 시그널 체인 모듈 식별 **only** |

**금지:**

- 악센트 색을 앰버 말고 추가하지 말 것. 브랜드 악센트는 하나다.
- `--color-*`(블록 타입색)을 **버튼·링크·포커스·에러 등 인터랙션에 쓰지 말 것.**
  이것들은 12개 GP-150 모듈을 눈으로 구분하기 위한 길찾기 색이다.
  (과거 코드가 `--color-amp` 를 링크색, `--color-dly` 를 포커스 링, `--color-rvb` 를
  에러색으로 쓰고 있었다 — 전부 규칙 위반이라 걷어냈다.)
- 네온·형광 채도 금지. `--glow-*` 는 4px 소프트 블룸 한 겹까지. 이중 헤일로 금지.
- 임의 hex 금지. 새 색이 필요하면 먼저 토큰을 늘리고 이유를 여기 적을 것.

**대비 불변 제약** (`web/lib/__tests__/tokens.test.ts` 가 자동 검증):
블록 타입 악센트는 패널과 명도차 ≥0.20, 배지 텍스트 대비 ≥4.5:1,
`--text`/`--text-muted`/`--lcd-text` 도 각 배경 대비 ≥4.5:1.

## 2. 타이포

| 패밀리 | 토큰 | 용도 |
| --- | --- | --- |
| Oswald (콘덴스드) | `--font-display` | 디스플레이 · **모든 실크스크린 라벨** · 버튼 · 배지 · 섹션 헤더 |
| Barlow | `--font-body` | 본문 · 설명문 |
| Space Mono | `--font-mono` | **모든 수치·유닛·모델명·LCD 문자열** |

- 라벨·버튼·배지·섹션 헤더 = **대문자 + `--tracking-label`(0.14em)**.
  한글은 케이스가 없으므로 자간과 콘덴스드 서체로 각인감을 낸다.
- 본문은 문장 케이스.
- **노브 값·수치는 항상 모노 + 단위.** 모호 표현 금지 (`320ms`, `64%`, 안 되면 숫자로).
- 폰트 로딩은 `app/layout.tsx` 의 `next/font` (self-host + `display:swap`).
  외부 `@import` 금지 — 렌더 블로킹.

## 3. 릴리프 — 이 언어의 핵심 기제

깊이는 **두 방향뿐**이다.

- **융기(raised)** — 패널·버튼·노브. `--shadow-panel` / `--shadow-raised` /
  `--shadow-button` / `--shadow-knob`. 드롭 섀도 + 상단 인너 하이라이트.
- **함몰(recessed)** — 웰·인풋·미터·페이더 슬롯. `--shadow-well` (inset).

카드는 "떠 있는 카드"가 아니라 **섀시 패널**이다: 금속면(`--metal-panel`) +
헤어라인 보더(`--border-metal`) + 융기 그림자 + (선택) 코너 나사 +
실크스크린 헤더 + 헤어라인 구분선.

글래스모피즘·backdrop-blur 금지. 하드웨어는 불투명하다.

## 4. 모양 · 간격

- 라운드는 타이트하게: `--radius-sm` 3px(칩) / `--radius-md` 5px(버튼·인풋) /
  `--radius-lg` 8px(패널) / `--radius-xl` 12px(섀시).
- 4px 그리드(`--space-*`). 컨트롤 간격 `--control-gap`, 패널 내부 `--panel-pad`.
- 보더는 얇고 어두운 금속 헤어라인 1px.

## 5. 모션

- 기계적이고 스냅. `--ease-mech` / `--ease-click`, `--dur-instant|fast|base`.
- 버튼 누름 = **2px 내려앉고 `--shadow-button-press` 로 뒤집힘**. 이게 가장 강한 피드백.
- 긴 페이드·바운스·패럴랙스 금지.
- `transform` / `opacity` / `box-shadow` 만 애니메이션(컴포지터 친화).
  `width`·`height`·`top`·`margin`·`font-size` 애니메이션 금지.
- `prefers-reduced-motion: reduce` 는 `app/globals.css` 가 전역 처리.

## 6. 아이콘 · 글리프

- **전용 아이콘 세트 없음.** 이 브랜드의 "아이콘"은 하드웨어 컨트롤 자체다
  (노브·스위치·LED·미터·페이더) — 글리프가 아니라 컴포넌트.
- **이모지 절대 금지.**
- 기능적 유니코드 마크만 허용: `→`/`›`(시그널 흐름), `▼`(셀렉트 캐럿), `⤓`(저장).
- 글리프 세트가 정말 필요해지면 Lucide 로 통일하고 이 문서에 기록할 것.

## 7. 카피 톤

- 확신 있고 손에 잡히는, 기어를 아는 목소리. "사용자"가 아니라 **연주자**에게 말한다.
- 동사 우선 명령형: *톤 생성, 패치 불러오기, 다이얼 맞추기*.
- 실제 기어 어휘를 쓴다: 시그널 체인, 블록, 바이패스, 드라이브, 캐비닛, 헤드룸.
- 마케팅 뻥 금지. 파라미터는 스펙 시트처럼 정확하게.
- 문서·UI 카피는 한국어, 실크스크린 라벨은 영문 대문자를 섞어도 된다(실제 기어가 그렇다).

## 8. 프리미티브 — 새로 만들지 말고 이걸 써라

`web/lib/hardware.css` 의 전역 `.tf-*` 클래스. DS 의 JSX 프리미티브를 1:1 이식한 것.

| 클래스 | DS 원본 | 용도 |
| --- | --- | --- |
| `.tf-panel` (+`--well`, `__screws`, `__head`, `__title`) | `Panel.jsx` | 섀시 패널 |
| `.tf-btn` (+`--primary/secondary/ghost/danger`, `--sm/lg`) | `Button.jsx` | 눌리는 버튼 |
| `.tf-field` / `.tf-input` / `.tf-select` | `Input.jsx` `Select.jsx` | 함몰 폼 컨트롤 |
| `.tf-badge` (+`--accent/success/danger/info`) | `Badge.jsx` | 실크스크린 칩 |
| `.tf-led` (+`--on`, `--green/red/blue`) | `LED.jsx` | 상태 렌즈 |
| `.tf-silkscreen` | — | 각인 라벨 유틸 |

**릴리프 레시피를 `*.module.css` 에 복붙하지 말 것.** 프리미티브에 없으면
`hardware.css` 에 추가하고 이 표를 갱신한다. 화면 고유 레이아웃만 모듈 CSS에 둔다.

## 9. 레이아웃

- 상단에 고정 섀시 바(`components/layout/Header.tsx`) — 워드마크 + 전원 LED + 내비 + 기기 배지.
- 시그널 체인은 데스크톱에서 가로 스크롤, 모바일에서 세로 스택.
- 반응형 검증 폭: 320 · 375 · 768 · 1024 · 1440. 가로 오버플로 0.

## 10. 접근성 (양보 없음)

- 포커스는 항상 보인다 — 앰버 2px 아웃라인 + 2px 오프셋(전역 `:focus-visible`).
- 터치 타깃 최소 44px.
- 색만으로 의미를 전하지 않는다. LED 옆에는 항상 텍스트 라벨이 있다.
- 장식용 요소는 `aria-hidden`(나사·LED 등).
- 대비 최소 4.5:1 — 토큰 테스트가 회귀를 막는다.

## 11. 의도적 편차 (DS 와 다르게 간 곳 · 근거 포함)

DS 를 그대로 따르지 않은 지점은 여기 전부 적는다. 이유 없이 다른 건 버그다.

1. **시그널 체인은 가로 스크롤이 아니라 wrap.** DS readme 는 "데스크톱에서 가로 스크롤"이라
   했지만, DS 의 그 문장은 상단 트랜스포트 바가 있는 **에디터**를 전제한다. 이 앱의 곡 상세는
   조작이 아니라 **판독** 화면이라, 가로 스크롤은 체인 뒷부분을 숨기고 키보드 접근성을 떨어뜨린다.
   `flex-wrap: wrap` 으로 전체 체인을 한눈에 보이게 유지한다.
2. **노브는 읽기 전용.** DS `ChainKnob` 은 드래그로 값을 바꾸지만 카탈로그는 저장된 패치를
   보여주는 화면이라 조작 핸들러가 없다. 눈금 21개 span 대신 conic-gradient 링 하나로
   같은 그림을 만든다(DOM 노드 수 ↓).
3. **밸류 아크는 점각이 아니라 얇은 링.** DS JSX 는 21개 점각 눈금이지만, `mask-composite` 로
   눈금을 새기면 브라우저별 합성 순서가 갈려 시작 지점에 통짜 쐐기가 남는다. DS readme 의
   문장("a subtle amber value-arc")을 따라 얇은 아크 한 겹으로 간다.
4. **unit 값은 노브가 아니라 LCD 칩.** `640ms` 처럼 상한이 데이터에 없는 값은 각도를 지어낼 수
   없다. 없는 정보를 그림으로 꾸며내는 대신 실제 기어처럼 숫자 창으로 띄운다.
5. **곡 제목은 대문자 변환하지 않는다.** 실크스크린 대문자 규칙은 **라벨**에만 적용된다.
   제목은 콘텐츠라 원 표기를 보존한다("Don't Look Back in Anger").
6. **`✕` (닫기)를 기능 마크로 허용.** DS 의 허용 목록(`→` `›` `▼` `⤓`)에는 없지만 DS 자신이
   "close 같은 글리프가 필요해지면 Lucide 로"라고 유보해 둔 항목이다. Lucide 도입 전까지 유지.
