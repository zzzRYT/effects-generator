# 프로세서 모델 — Valeton GP-150

재사용 가능한 멀티이펙터 모델 정의. 여러 기타에서 공통으로 쓰이므로 한 번만 정의하고 rig에서 참조한다. → [[rigs]]

세부 데이터는 같은 폴더의 분리 레퍼런스를 본다:

- [[hardware]] — 본체 패널·시그널 체인·EXP/CTRL·글로벌 EQ 운용
- [[amps]] — 앰프 모델 전체(60여 종) ↔ 기반 앰프 ↔ 노브 (핵심 데이터)
- [[cabs]] — 캐비넷 모델 전체 ↔ 기반 스피커 ↔ 노브
- [[effects]] — 이펙트 블록 전체(NR·PRE·WAH·DST·EQ·MOD·DLY·RVB·VOL) ↔ 기반 페달 ↔ 노브

## 개요

- **모델링**: 2세대 HD 모델링, 200+ 이펙트
- **모듈**: 최대 12개 동시, 순서 자유 배치
- **캐비닛**: IR 로딩(서드파티 20개) + NAM(SnapTone, 팩토리 50 포함 최대 100)
- **컨트롤**: 풋스위치 2 + 익스프레션 페달 (→ 곡 중 사운드 전환에 활용)
- **기타**: 루퍼(180초), 드럼머신, 48kHz/24bit, USB 오디오 인터페이스

## 기본 시그널 체인 규칙

표준 배치 순서(특별한 이유 없으면 이대로):

```
기타 → [게이트] → [드라이브/부스트] → [앰프] → [캐비닛/IR]
     → [모듈레이션] → [딜레이] → [리버브] → 출력
```

- **게이트**: 하이게인일 때만 추가
- **드라이브/부스트**: 앰프 앞 (게인 푸시) 또는 솔로 부스트용
- **공간계(딜레이/리버브)**: 항상 체인 뒤쪽
- **모듈레이션**(코러스/플랜저)는 보통 앰프 뒤, 딜레이 앞

레퍼런스는 V1.0.5 영문 온라인 매뉴얼(PDF)에서 추출·정리했다.
